Accessibility Statement

Last updated: April 3, 2026

Our commitment

Swipcall is committed to ensuring digital accessibility for people with disabilities. We continually improve the user experience for everyone and apply the relevant accessibility standards to this website and our Android application.

Standards

This website aims to conform to:

  • WCAG 2.1 Level AA — Web Content Accessibility Guidelines
  • EN 301 549 — European standard for ICT accessibility
  • European Accessibility Act (EAA) — Directive (EU) 2019/882

Measures taken

  • Semantic HTML structure with proper heading hierarchy
  • Sufficient color contrast ratios (WCAG AAA for text on backgrounds)
  • Keyboard navigation support throughout the site
  • Skip-to-content link for keyboard users
  • ARIA labels on interactive elements and icons
  • Responsive design that supports zoom up to 200%
  • Respect for user motion preferences (prefers-reduced-motion)
  • Focus indicators on all interactive elements
  • On-site accessibility tool allowing font size, line height, word spacing, contrast, and display adjustments
  • Focus trapping in modal dialogs for assistive technology users
  • Ability to pause all animations

Accessibility tool

This website includes a built-in accessibility tool, accessible via the orange button at the bottom-left corner of every page. The tool provides:

  • Text adjustments: Font size scaling, line height, word spacing, readable font (dyslexia-friendly)
  • Display adjustments: High contrast mode, grayscale mode, link highlighting
  • Navigation: Large cursor, pause all animations
  • All settings persist across pages and sessions
  • Reset all adjustments with one click

Android application

The Swipcall Android app is designed with accessibility in mind:

  • Minimum touch targets of 48x48dp for all interactive elements
  • High contrast dark theme with WCAG AAA compliant color ratios
  • Content descriptions on all buttons and icons for screen readers
  • Support for Android system font scaling up to 200%
  • Respects Android "reduce motion" system setting

Known limitations

While we strive for full accessibility, some areas may have limitations:

  • The post-call popup overlay relies on Android system overlay permissions, which may interact differently with assistive technologies depending on the device and Android version.
  • Some decorative SVG illustrations on this website may not convey meaning to screen readers, though they are marked as decorative with appropriate ARIA attributes.
